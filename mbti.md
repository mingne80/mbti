# MBTI 검사 웹페이지 기획 · 설계 문서


## 1. 개요


부서 인원이 닉네임을 입력하고 접속하여 MBTI 검사를 수행하고, 결과를 미니 DB에 저장한 뒤, 모든 접속자가 함께 볼 수 있는 대시보드에서 구성원들의 MBTI 분포를 확인할 수 있는 웹페이지.


### 운영 목표


- **대상 규모**: 부서원 약 20명 (소규모 · 동시접속 부하 낮음)


- **배포 방식**: 무료 공개 호스팅에 간단히 배포하여 URL로 공유


- **전제**: 서버리스 무료 호스팅 환경에서도 데이터가 영구 저장되어야 함 → 파일 기반 DB(SQLite 로컬 파일)는 부적합, **무료 티어 관리형 DB** 사용


### 핵심 사용자 흐름


```


[닉네임 입력]  →  [MBTI 검사 진행]  →  [결과 확인]  →  [DB 저장]


                                                        │


                                                        ▼


                                    [대시보드 - 전체 구성원 결과 + 도넛 그래프]


```

## 2. 요구사항 정리


| 구분 | 요구사항 |


|------|----------|


| 입력 | 접속 시 닉네임 입력 |


| 검사 | 인터넷에서 보편적으로 통용되는 MBTI 검사 문항으로 구성 |


| 결과 | 4가지 지표(E/I, S/N, T/F, J/P) 조합으로 16유형 산출 |


| 저장 | 닉네임 + MBTI 결과를 미니 DB에 저장 |


| 공유 | 모든 접속자가 볼 수 있는 별도 대시보드 페이지 |


| 시각화 | 부서 인원의 MBTI 경향성을 **도넛 그래프**로 표시 |

## 3. 기술 스택 (권장)


무료 공개 호스팅 + 20명 규모를 전제로, **전부 무료 티어로 운영 가능하며 서버리스에서도 데이터가 영구 저장되는** 조합을 선택.


| 영역 | 기술 | 선정 이유 |


|------|------|-----------|


| 프레임워크 | **Next.js (App Router, React 18)** | 프론트+백엔드(API Route) 통합, Vercel 무료 배포에 최적화 |


| 언어 | **TypeScript** | 타입 안정성, 유지보수 용이 |


| 스타일 | **Tailwind CSS** | 빠른 UI 구성 |


| 미니 DB | **Turso (libSQL / SQLite 호환)** | 무료 티어 제공, SQLite 문법 그대로, 서버리스에서 영구 저장 |


| DB 클라이언트 | **@libsql/client** | Turso 접속용 경량 클라이언트 (또는 Drizzle ORM) |


| 차트 | **Chart.js + react-chartjs-2** | 도넛(Doughnut) 차트 기본 지원, 가벼움 |


| 배포 | **Vercel (무료 Hobby 플랜)** | Git 연동 자동 배포, HTTPS URL 즉시 발급, 20명 규모 무료 한도 내 |


### 무료 호스팅 · DB 조합 비교


| 조합 | 호스팅 | DB | 특징 |


|------|--------|----|------|


| **① 권장** | Vercel | Turso (libSQL) | SQLite 문법 유지 + 서버리스 영구저장, 설정 간단 |


| ② | Vercel | Supabase / Neon (Postgres) | 정통 관계형 DB, 무료 티어 넉넉, Postgres 문법 |


| ③ | Vercel | Vercel Postgres / KV | Vercel 대시보드에서 바로 연결, 무료 한도 존재 |


> **왜 로컬 SQLite 파일을 쓰지 않는가**: Vercel·Netlify 등 서버리스 무료 호스팅은 배포마다 파일시스템이 초기화되고 쓰기가 유지되지 않음. 따라서 파일 DB에 저장한 결과가 사라짐 → 20명이 함께 쌓는 데이터는 **외부 관리형 DB(Turso/Postgres)** 에 저장해야 함.


> Turso는 SQLite 호환이라 앞서 설계한 SQL 스키마를 거의 그대로 사용 가능. 로컬 개발 시에는 SQLite 파일, 배포 시에는 Turso로 연결 URL만 교체하는 구성도 가능.


### 대안 (더 간단하게)


- 백엔드 없이: **Vite + React + LocalStorage** → 여러 사람 공유 대시보드 불가하므로 부적합


- 공유가 핵심 요구사항이므로 **서버 + 중앙 DB(관리형) 구조 필수**

## 4. 페이지 구성


### 4.1 `/` — 닉네임 입력 (시작 페이지)


- 닉네임 입력 폼


- "검사 시작" 버튼 → `/test` 로 이동 (닉네임 상태 전달)


- 중복 닉네임 처리: 허용하되 결과 저장 시 새 레코드 생성 (또는 덮어쓰기 정책 선택)


### 4.2 `/test` — MBTI 검사


- 문항을 한 문제씩 또는 리스트로 표시


- 각 문항은 4개 지표 중 하나에 가중치 부여


- 진행률 표시 바


- 모든 문항 응답 후 "결과 보기" → 점수 집계 → `/result`


### 4.3 `/result` — 결과 확인 & 저장


- 산출된 MBTI 유형(예: `ENFP`) + 유형별 간단 설명 표시


- 자동으로 API 호출하여 DB 저장


- "대시보드 보기" 버튼 → `/dashboard`


### 4.4 `/dashboard` — 공용 대시보드 (전원 조회 가능)


- **상단**: 전체 구성원 목록 테이블 (닉네임 · MBTI · 검사일시)


- **하단**: 부서 인원 MBTI 경향성 **도넛 그래프**


  - 옵션 A: 16유형별 분포 도넛


  - 옵션 B(권장): 지표별 경향성 도넛 4개 (E vs I, S vs N, T vs F, J vs P)


  - 실시간 반영 (주기적 갱신 또는 새로고침)

## 5. 데이터 설계


### 5.1 DB 스키마 (Turso / SQLite 호환)


```sql


CREATE TABLE IF NOT EXISTS results (


  id          INTEGER PRIMARY KEY AUTOINCREMENT,


  nickname    TEXT    NOT NULL,


  mbti        TEXT    NOT NULL,          -- 예: 'ENFP'


  scores      TEXT,                       -- JSON: {"E":6,"I":2,...} (선택 저장)


  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP


);


```


> Turso는 SQLite 호환이므로 위 스키마를 그대로 사용. 접속은 환경변수(`TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`)로 관리하고 Vercel 프로젝트 설정에 등록.


### 5.2 API 설계 (Next.js Route Handler)


| Method | 경로 | 설명 |


|--------|------|------|


| `POST` | `/api/results` | 닉네임+MBTI 결과 저장 |


| `GET`  | `/api/results` | 전체 결과 목록 조회 (대시보드용) |


| `GET`  | `/api/stats`   | 지표별/유형별 집계 통계 (도넛 차트용) |


**POST `/api/results` 요청 예시**


```json


{ "nickname": "홍길동", "mbti": "ENFP", "scores": {"E":7,"I":1,"S":3,"N":5,"T":2,"F":6,"J":4,"P":4} }


```


**GET `/api/stats` 응답 예시**


```json


{


  "byType": { "ENFP": 3, "ISTJ": 5, "...": 0 },


  "byAxis": {


    "EI": { "E": 12, "I": 8 },


    "SN": { "S": 9,  "N": 11 },


    "TF": { "T": 7,  "F": 13 },


    "JP": { "J": 10, "P": 10 }


  }


}


```

## 6. MBTI 검사 문항 설계


인터넷에서 보편적으로 통용되는 4지표 기반 문항으로 구성. **지표당 3문항, 총 12문항** (간이형) 또는 **지표당 5~6문항, 총 20~24문항** (표준형) 권장.


### 6.1 지표별 문항 구조


- **E(외향) / I(내향)**: 에너지 방향


- **S(감각) / N(직관)**: 정보 인식


- **T(사고) / F(감정)**: 판단 기준


- **J(판단) / P(인식)**: 생활 양식


### 6.2 문항 예시 (5점 척도 또는 양자택일)


각 문항은 "매우 그렇다 ~ 매우 아니다" 5점 척도 또는 A/B 선택형.


**E vs I**


1. 처음 만나는 사람과도 쉽게 대화를 시작하는 편이다. → E


2. 여러 사람과 어울릴 때보다 혼자만의 시간에 에너지가 충전된다. → I


3. 모임에서 주로 먼저 말을 거는 편이다. → E


**S vs N**


4. 구체적인 사실과 세부 사항에 주목하는 편이다. → S


5. 큰 그림과 미래의 가능성을 먼저 상상한다. → N


6. 설명은 실제 경험과 예시로 이해하는 것이 편하다. → S


**T vs F**


7. 결정을 내릴 때 논리와 객관적 사실을 우선한다. → T


8. 다른 사람의 감정과 상황을 먼저 고려한다. → F


9. 갈등 상황에서 옳고 그름을 따지는 편이다. → T


**J vs P**


10. 계획을 세우고 그대로 실행하는 것을 선호한다. → J


11. 상황에 따라 유연하게 대응하는 것이 편하다. → P


12. 할 일을 미리 끝내야 마음이 편하다. → J


> 실제 구현 시 문항은 JSON/상수 파일(`questions.ts`)로 분리하여 관리.


### 6.3 채점 로직


```


각 지표별로 응답 점수 합산 →


  E점수 > I점수 → 'E', 아니면 'I'


  (S/N, T/F, J/P 동일)


→ 네 글자 조합 = 최종 MBTI


동점 시 정책: 사전 지정된 기본값(예: I, N, F, P) 또는 재질문


```

## 9. 개발 단계 (권장 순서)


1. **프로젝트 초기화** — `npx create-next-app` (TS, Tailwind)


2. **Turso DB 준비** — 무료 계정 생성 → DB 만들기 → 접속 URL·토큰 발급 → 테이블 생성


3. **DB 연결** — `@libsql/client`로 `lib/db.ts` 작성, 환경변수(`.env.local`) 사용


4. **문항 데이터 작성** — `questions.ts`


5. **닉네임 입력 페이지** — 상태 관리


6. **검사 페이지** — 문항 렌더링 + 채점 로직


7. **결과 페이지** — 유형 산출 + API 저장


8. **대시보드** — 목록 조회 + 도넛 차트


9. **스타일 · 반응형 정리**


10. **배포** — GitHub 저장소 push → Vercel 연동 → 환경변수 등록 → 자동 배포 → 발급된 URL 공유


### 배포 절차 요약 (무료)


1. Turso 가입 후 DB 생성, `TURSO_DATABASE_URL` / `TURSO_AUTH_TOKEN` 확보


2. 코드를 GitHub(공개/비공개 무관)에 push


3. Vercel 가입 → GitHub 저장소 Import


4. Vercel 프로젝트 환경변수에 Turso URL·토큰 등록


5. Deploy → `https://프로젝트명.vercel.app` 형태의 URL을 부서원에게 공유

## 10. 추가 고려사항


- **개인정보**: 닉네임만 수집(실명 지양), 사내 용도 명시


- **공개 호스팅 접근 제어**: URL이 공개되므로 외부인 유입 가능. 20명 규모이면 (a) 간단한 공용 입장 암호 한 개, 또는 (b) 페이지 하단 "부서 내부용" 안내로 충분. 민감정보 없으므로 과한 인증 불필요


- **중복 검사**: 동일 닉네임 재검사 시 덮어쓰기 vs 누적 정책 결정 필요 (닉네임 UNIQUE 제약 + UPSERT로 덮어쓰기 권장 → 대시보드 중복 방지)


- **무료 티어 한도**: Vercel Hobby·Turso 무료 티어 모두 20명 규모 트래픽·저장량에 충분. 상업적 사용은 약관 확인 필요


- **결과 신뢰성 고지**: "본 검사는 재미·참고용이며 공식 MBTI가 아님" 문구 표기 권장


- **초기 데이터**: 대시보드가 비어 보이지 않도록 안내 문구/빈 상태 UI 처리


- **비용**: 위 조합은 전부 무료 티어로 운영 가능 (도메인 커스터마이징만 유료 선택)
